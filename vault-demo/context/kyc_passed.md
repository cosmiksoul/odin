# KYC Passed

**Owner:** Compliance · **Источник данных:** KYC / Compliance

## Что это и зачем

Количество верифицированных игроков. Определяет потолок для выводов и полного доступа к продукту

## Формула — детали

Числитель: Count KYC completed

Знаменатель: —

## Ловушки

1) KYC pass rate сильно различается по ГЕО: EU 80-90%, India 60-70%, Bangladesh 40-50% (качество документов)
2) Низкий KYC pass rate может быть UX проблемой (плохие инструкции, не те форматы), не фрод
3) KYC pre-FTD снижает Reg2FTD на 15-30%. Считать компромисс комплайнс vs конверии

## Как улучшать

1) Автоматизация: OCR + проверку живости + API verification (Sumsub, Shufti, Onfido)
2) KYC после FTD (где разрешено): не блокировать конверсию
3) Progressive KYC: минимум для депозита, полный для вывода >порога
4) Понятный UX: пошаговый прогресс, примеры поддерживаемых документов
