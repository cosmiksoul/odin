# Event Quality Rate

**Owner:** Analytics · **Источник данных:** Monitoring

## Что это и зачем

Доля событий с корректными и полными свойствами. Событие пришло, но в нём мусор вместо данных, это хуже, чем если бы оно не пришло. Качество данных определяет качество решений.

## Формула — детали

Числитель: Valid events

Знаменатель: Total events

## Ловушки

1) Цель: >99% для финансовых событий, >95% для поведенческих. 2) Качество деградирует после деплоев, проверяйте данные после каждого релиза. 3) Некоторые null-значения легитимны, не все пропуски баги, иногда данных просто нет.

## Как улучшать

1) Валидация схемы на этапе приёма, битое событие не попадает в хранилище. 2) Автоматические тесты качества данных, как юнит-тесты, но для данных. 3) Event contracts, формальное соглашение «вот такие поля в таком формате». 4) Real-time quality dashboard, видим деградацию сразу.
