# Instrumentation Coverage

**Owner:** Analytics · **Источник данных:** Analytics

## Что это и зачем

Покрытие ключевых пользовательских действий аналитикой. Если событие не трекается, оно для вас не существует. Нельзя оптимизировать то, что не измеряешь. Это фундамент всех data-driven решений.

## Формула — детали

Числитель: Instrumented flows

Знаменатель: Total key flows

## Ловушки

1) Цель: 100% критических событий, 80%+ вторичных. 2) Покрытие без точности, самообман: событие трекается, но данные в нём кривые. 3) Чрезмерная инструментация создаёт шум, трекать всё подряд не нужно, фокус на actionable-данных.

## Как улучшать

1) Event taxonomy: список всех критических событий, от регистрации до вывода. 2) Автоматические проверки покрытия при каждом релизе, чтобы новая фича не осталась без трекинга. 3) Ежеквартальный аудит покрытия. 4) Гайдлайны для разработчиков, какие события и как трекать.
