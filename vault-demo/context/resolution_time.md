# Resolution Time

**Owner:** Support · **Источник данных:** Support

## Что это и зачем

Время от создания тикета до полного решения. Показывает реальную скорость поддержки, не когда ответили, а когда реально помогли. Долгое решение = зависший вывод, незаконченная верификация, злой игрок

## Формула — детали

Числитель: Median(resolution − ticket_created)

Знаменатель: —

## Ловушки

1) Таргеты по типу: <4 часов для платёжных вопросов, <24 часов для общих, <48 часов для сложных
2) Resolution time без CSAT, неполная картина: быстро закрыли, но игрок недоволен, это не решение
3) Выбросы сильно искажают среднее: один тикет, висевший 2 недели, портит всю статистику. Использовать медиану

## Как улучшать

1) Knowledge base для агентов: быстрый доступ к решениям типовых проблем
2) Чёткие escalation paths: кто решает что, без «пинг-понга» между отделами
3) Авто-решение рутинных задач: сброс пароля, активация бонуса, статус вывода, без участия человека
4) Чинить корневые причины: если тикеты возвращаются, проблема в продукте, а не в саппорте
