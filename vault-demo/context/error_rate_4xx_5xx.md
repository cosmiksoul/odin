# Error Rate (4xx/5xx)

**Owner:** Tech · **Источник данных:** Monitoring

## Что это и зачем

HTTP-ошибки, индикатор стабильности платформы. 5xx, сервер сломался, 4xx, клиент отправляет что-то не то (или бот стучится). Рост ошибок = деградация пользовательского опыта.

## Формула — детали

Числитель: Error responses

Знаменатель: Total requests

## Ловушки

1) 5xx больше 0.1% = немедленная реакция, это серверные сбои. 2) 4xx раздуваются ботами, фильтруйте при анализе. 3) Критичен мониторинг по отдельным эндпоинтам, общий показатель маскирует проблемы конкретных API.

## Как улучшать

1) Мониторинг ошибок: Sentry, Datadog, видеть каждую ошибку с контекстом. 2) Error budget по принципам SRE, запас ошибок, который нельзя превышать. 3) Авто-откат при провале health check, не ждать, пока починят. 4) Разделяйте клиентские (4xx) и серверные (5xx), это разные проблемы.
