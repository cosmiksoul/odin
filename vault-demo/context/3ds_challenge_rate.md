# 3DS Challenge Rate

**Owner:** Payments · **Источник данных:** Payments

## Что это и зачем

Доля карточных транзакций, которые уходят на 3DS challenge (дополнительную верификацию). Каждый challenge, это всплывающее окно банка, где игрок может отвалиться. Минус 10-15% конверсии на каждый challenge, ощутимо

## Формула — детали

Числитель: 3DS challenged txns

Знаменатель: Total card deposits

## Ловушки

1) Challenge rate сильно различается по регионам: EU 40-60% (PSD2 обязывает), за пределами EU 5-15%
2) Challenge rate = 0%, скорее всего 3DS вообще не внедрён, а не идеальная оптимизация. Это риск по чарджбэкам
3) Высокий challenge rate + высокий auth success = нормально. Проблема, когда challenge rate высокий, а auth после challenge низкий

## Как улучшать

1) 3DS2 frictionless flow: передавать максимум данных (device fingerprint, IP, историю транзакций): чем больше данных, тем чаще банк пропускает без challenge
2) TRA exemption (освобождение по анализу рисков) для транзакций до 30 EUR, легальный способ обойти challenge
3) Whitelisting-программы: постоянный игрок может добавить оператора в trusted merchants у своего банка
4) Качество device fingerprint: чем лучше сбор данных об устройстве, тем чаще frictionless path
