# Bonus Abuse Cluster Count

**Owner:** Risk · **Источник данных:** Risk / Antifraud

## Что это и зачем

Количество выявленных кластеров бонус-абьюза (групп связанных аккаунтов, работающих совместно). Не просто «сколько абьюзеров», а «сколько организованных схем», это масштаб проблемы

## Формула — детали

Числитель: Count of linked account clusters

Знаменатель: —

## Ловушки

1) Zero clusters = система детекции не работает, а не «нет абьюза». Всегда проверять методологию
2) Shared household = false positive: семья на одном Wi-Fi, это не кластер. Нужна дополнительная валидация
3) Detection latency matters: кластер, найденный через месяц, когда деньги уже выведены, бесполезен

## Как улучшать

1) Device fingerprinting: один телефон/браузер, несколько аккаунтов = кластер
2) Behavioral patterns: одинаковое время входа, одни игры, похожие суммы
3) Bet analysis: ставки на противоположные исходы с разных аккаунтов = matched betting
4) Auto restrictions: при обнаружении кластера, блокировка бонусов для всей группы
