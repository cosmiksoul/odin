# Cohort Completeness

**Owner:** Analytics · **Источник данных:** BI

## Что это и зачем

Полнота данных когорт для анализа. Если в когорте не хватает 20% данных по депозитам, ваш LTV занижен. Неполные когорты = кривые выводы = неправильные решения по бюджетам и стратегии.

## Формула — детали

Числитель: Complete cohorts

Знаменатель: Total cohorts

## Ловушки

1) 20% пропущенных данных по депозитам = LTV занижен, а вы об этом не знаете. 2) Полнота деградирует со временем, старые когорты «теряют» данные из-за миграций и ротации. 3) Кросс-платформенные когорты требуют identity resolution, без него один игрок = два «человека».

## Как улучшать

1) Мониторинг data pipeline, ловим пропуски автоматически. 2) Процедуры backfill, умеем «залатать» пробелы в данных. 3) Консистентное определение когорт, одна когорта = одна формула, без разночтений. 4) Стратегия imputation для пропущенных данных, чем заполнять пробелы.
