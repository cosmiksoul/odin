# Payment Method Mix

**Owner:** Payments · **Источник данных:** Payments

## Что это и зачем

Распределение депозитов по методам оплаты: карты, e-wallets, банковские переводы, крипта, локальные методы. Показывает, чем платит аудитория, и определяет cost structure платёжной инфраструктуры

## Формула — детали

Числитель: Deposits per payment method

Знаменатель: Total deposits

## Ловушки

1) Volume ≠ value: крипта может быть 5% транзакций, но 30% по сумме, один крипто-депозит заменяет десять карточных
2) Mix сдвигается при изменении user base: пришли из нового гео, весь mix поехал. Смотреть по когортам
3) Некоторые методы несут повышенный fraud: prepaid-карты и определённые e-wallets, выше риск чарджбэков

## Как улучшать

1) Добавлять локальные методы под каждое гео: PIX для Бразилии, UPI для Индии, iDEAL для Нидерландов, без них теряешь аудиторию
2) Smart routing: автоматически выбирать оптимальный PSP для каждого метода
3) Default method по гео и устройству: показывать первым то, чем аудитория реально платит
