# Login Success Rate

**Owner:** Tech / Security · **Источник данных:** Auth

## Что это и зачем

Надёжность системы аутентификации. Если игрок не может залогиниться, он не может играть и платить. Всё просто: нет логина = нет дохода. Плюс это первое впечатление при каждом визите.

## Формула — детали

Числитель: Successful logins

Знаменатель: Login attempts

## Ловушки

1) Ниже 95% = проблема с аутентификацией, нужно разбираться. 2) Разделяйте «неправильный пароль» и техническую ошибку, это разные проблемы. 3) Всплеск неудачных логинов = возможная brute-force атака, мониторьте.

## Как улучшать

1) Несколько способов входа: пароль, соцсети, биометрия, что удобнее. 2) Простое восстановление пароля, без квестов. 3) «Запомнить меня», минимизируем необходимость повторного входа. 4) SSO (Single Sign-On) между продуктами, один логин для всего.
