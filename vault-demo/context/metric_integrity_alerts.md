# Metric Integrity Alerts

**Owner:** Analytics · **Источник данных:** Monitoring

## Что это и зачем

Автоматические алерты на аномалии в ключевых метриках. Ваша система раннего предупреждения. Без них вы узнаёте о проблеме, когда игроки уже жалуются, а к тому моменту ущерб уже нанесён.

## Формула — детали

Числитель: Count(integrity alerts)

Знаменатель: Week

## Ловушки

1) Слишком много алертов = усталость, цель, 1–3 в день. 2) Пропущенный алерт хуже ложного, лучше перебдеть. 3) У каждого алерта должен быть owner, алерт без ответственного = просто шум.

## Как улучшать

1) Статистическое обнаружение аномалий: Z-score, IQR, ML-модели. 2) Мульти-метрическая корреляция, одна аномалия может быть шумом, две связанные, уже сигнал. 3) Многоуровневые алерты: warning → critical → emergency. 4) Подавление алертов для известных событий, промо-день не должен триггерить панику.
