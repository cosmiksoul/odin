## TC-2: Карточки метрик — рендер и контраст заголовков

**Цель:** проверить контрастный фикс задачи 8 (Concern C1 в code-review).

**Шаги:**
1. На открытой странице (Catalog view) посмотреть на сетку карточек
2. Подсчитать примерное количество карточек (должно быть ~47)
3. Прочитать названия первых 5 карточек подряд (например, `GGR Margin / Hold %`, `FTD Count`, `Uptime`...)
4. Сравнить визуально с твоими старыми скриншотами прототипа (где заголовки сливались с фоном)

**Ожидаемое:**
- Видны ~47 карточек
- Названия метрик читаются **без напряжения** — белым жирным шрифтом на тёмном фоне
- Контраст явно лучше, чем на старых скриншотах из прототипа

**Если PASS:** хорошо, фикс сработал. Концерн C1 закрыт.
**Если FAIL** (всё ещё плохо читается): отметь это явно — значит root cause НЕ в цвете (твоя гипотеза про неправильный CSS подтверждается, либо Code прав про шрифты). Делаем fix-cycle.

**Факт:** Ок

## TC-11: Переключение темы DARK ↔ LIGHT

**Цель:** проверить, что theme-switcher работает после рефакторинга.

**Шаги:**
1. Найти в topbar кнопку `DARK` / `LIGHT`
2. Нажать
3. Проверить, что страница перерисовалась (фон, текст, контраст карточек)
4. Нажать обратно

**Ожидаемое:**
- Фон, текст, цвета карточек корректно меняются
- Заголовки карточек **читаемы в обеих темах** (важно: фикс из задачи 8 должен работать и для light, и для dark)

**Если в LIGHT-теме заголовки карточек невидимы:** это связано с C1 — фикс через `var(--l1)` мог сломать читаемость в светлой теме (в dark `--l1` = белый, в light может быть = чёрный или другой). Зафиксировать как failure, разберём.

**Факт:** Кстати, для лайт темы заголовок виден - он черный

**Статус:** PASS
