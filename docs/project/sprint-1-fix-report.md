## FIX-2: Drawer title — нечитаемый цвет на тёмной теме

### Что я нашёл

Прокрутил скриншоты прототипа в `src/uploads/`. На `Screenshot 2026-04-24 163807.png` — drawer для метрики "Total Winnings". Большой заголовок «Total Winnings» рендерится **тусклым серым цветом**, заметно темнее, чем подзаголовок «Casino / Games» под ним. То есть основной h1-заголовок drawer'а **темнее** маленького subtitle — это нелогично и совпадает с описанием пользователя «карточка/блок с чёрным текстом на чёрном фоне» в широком смысле.

### Root cause

В `styles.css` селектор `.drawer__title` (строка 899-902 до фикса):

```css
.drawer__title {
  font-size: 34px; line-height: 1.1; font-weight: 600;
  margin: 0 0 6px; letter-spacing: -0.02em;
}
```

**`color` не задан.** Цвет наследуется от `body { color: var(--fg) }` — на dark-теме это `#efefef` (почти-белый). По CSS-логике должно быть ярко.

Но на скрине заголовок рендерится сильно темнее `#efefef` — выглядит как `~#3a3a3a`. Точную причину я не могу установить из CLI без браузерного DevTools. Гипотезы (для tech debt):

- (а) `<h1>` без явного `color` + JetBrains Mono 600 weight 34px на Windows-рендеринге — может рендериться тонкими штрихами и **выглядеть** темнее чем цветовое значение пикселя
- (б) Какой-то браузер user-agent style на `<h1>` ужимает контраст в дарк-режиме
- (в) Шрифт не успевает загрузиться (`@import url('https://fonts.googleapis.com/...')` — внешняя зависимость) и фоллбэк рендерится иначе

Все три гипотезы — это поведение браузера/окружения, не CSS-баг как таковой. Но **зацепить** проблему можно явным указанием цвета.

### Что поправлено

Один селектор, одно property добавлено:

**Было** (`styles.css:899-902`):

```css
.drawer__title {
  font-size: 34px; line-height: 1.1; font-weight: 600;
  margin: 0 0 6px; letter-spacing: -0.02em;
}
```

**Стало:**

```css
.drawer__title {
  font-size: 34px; line-height: 1.1; font-weight: 600;
  margin: 0 0 6px; letter-spacing: -0.02em;
  color: var(--l1);
}
```

`--l1` — существующая переменная палитры:
- `theme--dark`: `--l1: #ffffff` (чисто-белый, ярче чем `--fg: #efefef`)
- `theme--light`: `--l1: #0a0a0a` (тот же, что и `--fg` в светлой теме — без регрессии)

Это **тот же приём, что Sprint 1 task 8 для `.card__name`** — единая логика «там где нужен максимально читаемый заголовок, явно ставим `var(--l1)`».

### Подтверждение

- `npm run build` — собирается без ошибок, новые хеши: `dist/assets/index-DnLNEFek.css` (CSS обновился) + `dist/assets/index-dvu8cNWt.js`. Это значит CSS-правка попала в сборку.
- **Визуальная проверка в обеих темах — на пользователе** (CLI-агент не может открыть браузер и переключить тему). Ожидаемо:
  - DARK: «Total Winnings» рендерится ярко-белым, заметно ярче подзаголовка
  - LIGHT: цвет — `#0a0a0a` (тот же что `--fg`), визуально идентично тому, что было до фикса

## Notes / Tech Debt

В процессе аудита нашёл несколько мест с потенциально низкой контрастностью на dark-теме. **Не правил** в этом fix-цикле (один баг = один селектор). Перечислены здесь для возможных будущих фиксов:

1. **`.mini--l2` и `.mini--l3` отсутствуют в CSS.** В `styles.css:1006-1007` определён только `.mini--l1` (чёрный текст на лаймовом фоне). Для L2/L3-метрик в Dependencies-секции drawer'а чипы рендерятся с базовыми `.mini` стилями (без background, без color) — текст наследует `var(--fg)` от `.depchip`, выглядит как «полу-стилизованный» чип. Не «чёрный на чёрном», но визуальная неконсистентность. Аналог `.tag--l1/l2/l3` в строках 798-800 определён полностью — здесь явный пропуск дизайнера прототипа.

2. **`.bench__hdr` и `.deps__lbl` используют `var(--fg-4)` = `#555` на `.bench__col`/`.deps__col` фоне `var(--bg-2)` = `#111`** на dark-теме. Контрастность ~2.5:1 — ниже WCAG AA (4.5:1). Заголовки бенчмарков «TIER-1 · UK/Nordics/DACH» и подписи зависимостей «INPUTS · feeds this metric» читаются с трудом.

3. **`.snap__lbl`** (строка 912) — те же `var(--fg-4)` на `var(--bg-2)`. Подписи «CURRENT»/«WoW»/«MoM»/«14D TREND» в drawer'е едва видны. Но на скриншоте `163807.png` они выглядят более-менее (видимо потому что верхний регистр + letter-spacing помогают воспринимать).

4. **`.pinc__lvl--l3`** (строка 502) — `background: var(--l3); color: #000` → чёрный текст на тёмно-сером `#555` (на dark-теме). Это в **PinnedCard в графе** (sidebar, когда метрика запиннена), не в drawer. Если пользователь видел проблему именно там — это кандидат №1 на следующий фикс.

5. **Контрастность в LIGHT-теме** не проверял — все мои наблюдения про dark.

6. Принципиально: в `styles.css` нет ни одной CSS-rule, которая бы задала `.drawer__title` цвет вне фикса этого спринта. То есть до моего фикса заголовок drawer'а **во всех темах** наследовал цвет от `body`. То что на скриншоте (dark) он выглядит сильно темнее ожидаемого — это не CSS-баг с `color`, это поведение шрифта/рендера. Мой фикс ставит `var(--l1)` — это максимально яркий вариант из существующей палитры в dark. Если визуально не помогло — root cause не в `color`, а в шрифте, и тогда нужно решать через bundled-fonts (это отдельный спринт).
