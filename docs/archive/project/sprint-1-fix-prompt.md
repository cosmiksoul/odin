## Tasks

### FIX-1: `React is not defined` — пропущенный `React.useState` в `app.jsx`

**Симптом:** при переходе на CATALOG view браузер показывает пустой экран. В DevTools console:

```
Uncaught ReferenceError: React is not defined
    at FilterGroup (app.jsx:1678:27)
The above error occurred in the <FilterGroup> component:
    at FilterGroup
    at div
    at aside
    at FiltersDrawer
    at main
    at div
    at App
```

**Root cause:** в `src/app.jsx:1678` остался один пропущенный вызов `React.useState`:
```jsx
const [open, setOpen] = React.useState(() => { ... });
```

`app.jsx:1` импортирует только named-хуки: `import { useState, useMemo, useEffect, useRef } from 'react'`. Default-импорт `React` отсутствует — поэтому на этой строке `React` undefined → краш FilterGroup → краш всего CATALOG view (FilterGroup рендерится только на каталоге, поэтому Home работает).

**Что сделать:**
1. В `src/app.jsx:1678` — `React.useState(...)` → `useState(...)`. Импорт уже есть, ничего добавлять не надо.
2. **Безопасности ради:** прогнать `grep -n 'React\.' src/app.jsx src/graph-view.jsx` (можно через bash tool) и убедиться, что больше нет других пропущенных `React.X` в файлах с named-импортами. `tweaks-panel.jsx` НЕ трогать — там осознанный default-import + `React.X` стиль (см. ADR review).
3. После фикса — `npm run dev`, открыть localhost, кликнуть CATALOG, **визуально подтвердить** что страница рендерится и в console нет красных Error.

**Это 1 строка кода + sanity grep.** Не разворачивай в большой рефакторинг.

### FIX-2: Контраст текста в drawer — нечитаемый случай

**Симптом:** пользователь сообщает что в drawer (выезжающая панель справа при клике на карточку метрики) есть **карточка/блок с чёрным текстом на чёрном фоне**. Это отдельный кейс от заголовков карточек каталога (которые мы фиксили в Sprint 1 задача 8).

**Где смотреть:**
- Открыть `npm run dev`
- На главной (Home → Watchlist) кликнуть на любую красную карточку (например, GGR) — откроется drawer
- В drawer есть несколько подсекций: header (название метрики), теги (L1/Must/Daily/Source: BI), CUT BY переключатели, CURRENT/WoW/MoM значения, исторический график, и **дальше вниз** — Formula, Benchmarks, Common Traps, Dependencies (зависит от метрики)
- Прокрутить drawer вниз и **визуально найти** где именно текст не читается — это может быть один из этих блоков
- Часто такое случается с тегами/чипами: фон установлен, а текст наследует базовый чёрный — на тёмной теме сливается

**Что сделать:**
1. Запустить dev-server, найти конкретный кейс визуально
2. Открыть DevTools, через Inspector выяснить какой именно селектор имеет проблему
3. Поправить **только этот селектор** (по принципу task 8 из Sprint 1) — поднять контраст до читаемого, использовать существующие переменные палитры (`var(--fg)`, `var(--l1)`, `var(--l2)`, `var(--l3)` — см. `:root` в `styles.css`)
4. Проверить что фикс работает в **обеих темах** (DARK + LIGHT) — переключи тему через кнопку в topbar
5. В fix-report **указать конкретно** какой селектор был поправлен и какое было/стало значение

**ВАЖНО:** не переписывай CSS пачкой. Один кейс = один селектор. Если найдёшь несколько проблемных блоков — занеси все в Notes / Tech Debt, а в этом фиксе исправь только тот, на который пожаловался пользователь (наиболее заметный/повторяющийся).

## Замечание по git workflow (правило из CLAUDE.md, секция «Удаление файлов»)

В Sprint 1 ты удалил `Catalog.html` через `rm`, без `git rm`. Из-за этого git показывает его как `modified`, и при `git add .` (точкой) удаление не попало бы в индекс — пользователь должен помнить про `git add -A`. Я это уже добавил в CLAUDE.md как явное правило.

**В этом fix-цикле файлы не удаляются**, но впредь — если будешь удалять файлы, которые есть в HEAD: используй `git rm <file>`, не `rm <file>`. Это сохраняет git status чистым.
