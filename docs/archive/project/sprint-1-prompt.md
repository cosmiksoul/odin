# Sprint 1 — Build Foundation

## Overview

Превратить прототип Catalog в `src/` из «JSX через Babel-in-script» в нормальный фронтенд-проект с Vite + npm. Цель — `npm install && npm run dev` запускает рабочий каталог локально, `npm run build` собирает прод-бандл в `dist/`. Без визуальной регрессии: всё, что сейчас рисуется в `Catalog.html` через CDN-React, должно рисоваться идентично после билда. Без новых фич, без багфиксов, без изменений UX.

Это **первый спринт** в новом workflow. До него прототип запускался через `npx serve` / `python -m http.server` в режиме file://. Этот спринт даёт фундамент для всех последующих — UX-долги (Sprint 2), миграция данных в MD, DuckDB-слой, Electron-обёртка идут поверх работающего билда.

## Контекст прототипа (state перед спринтом)

В `src/` лежит:
- `Catalog.html` — entry point. Грузит React/ReactDOM/Babel-standalone из CDN, потом 4 JSX-файла как `<script type="text/babel">`
- `app.jsx` (2566 строк) — главный App + большинство компонентов карточки/каталога
- `graph-view.jsx` (502 строки) — Graph view
- `tweaks-panel.jsx` (419 строк) — общая панель твиков (UI-конструктор)
- `tweaks-ui.jsx` (24 строки) — конкретные твики для этого приложения
- `data.js` (555 строк) — `window.METRICS_SEED = [...]` с ~48 захардкоженными метриками
- `styles.css` (2333 строки) — кастомный CSS, без Tailwind/CSS-frameworks

**Архитектурные особенности**, которые нужно учесть при рефакторинге:
- Все JSX-файлы определяют **глобальные функции** (`function GraphView(...)`, `function TweaksPanel(...)`) — нет `import`/`export`. Компоненты ссылаются друг на друга через window-скоуп.
- В `app.jsx` есть деструктуризация `const { useState, useMemo, useEffect, useRef } = React;` — `React` берётся из глобального скоупа (CDN). В `graph-view.jsx` — повторная деструктуризация с переименованием (`useStateG` и т.д.) чтобы не пересекалось.
- Данные грузятся как `window.METRICS_SEED` — глобальная переменная.
- В `Catalog.html` есть `?v=3` / `?v=25` query-strings на стилях/скриптах — дешёвый cache-bust для прод-разработки. После билда не нужен.

## Tasks

### 1. Инициализация проекта

- В корне проекта (`C:\Users\cosmi\Projects\odin\`) создать `package.json` с:
  - **Зависимости:** `react@18.3.1`, `react-dom@18.3.1` (точно те же версии что в CDN-сборке прототипа, чтобы избежать сюрпризов поведения)
  - **DevDependencies:** `vite@^5`, `@vitejs/plugin-react@^4`
  - **Scripts:** `"dev": "vite"`, `"build": "vite build"`, `"preview": "vite preview"`
  - `"name": "odin"`, `"version": "0.1.0"`, `"private": true`, `"type": "module"`
- Создать `vite.config.js` с `@vitejs/plugin-react`, корнем `src/` (или с `root: 'src'`), и output `dist/` в корне проекта.
- Если выбран `root: 'src'` — entry-point `src/index.html` (переименовать из `Catalog.html`), либо настроить Vite так, чтобы `Catalog.html` оставался entry. **Решение по этому пункту на усмотрение Code** — главное чтобы `npm run dev` работал.

### 2. Рефакторинг JSX-файлов в ES-модули

Каждый из четырёх `.jsx` файлов превратить в нормальный ES-модуль:

- Убрать деструктуризацию `const { useState, useMemo, ... } = React`. Заменить на `import { useState, useMemo, ... } from 'react'`.
- В `graph-view.jsx` — алиасы (`useMemoG`, `useStateG`) можно убрать, после миграции на импорты конфликта имён больше не будет. Использовать обычные `useMemo`, `useState`.
- В каждом файле — `export` для публичных функций (главных компонентов).
- В `app.jsx` — `import` нужных компонентов из `graph-view.jsx`, `tweaks-panel.jsx`, `tweaks-ui.jsx`.
- ReactDOM.render / createRoot вызов — оставить в `app.jsx` (или вынести в отдельный `main.jsx` — на усмотрение Code, но если вынести, об этом написать в sprint-report).

### 3. Адаптация data.js

- Превратить `window.METRICS_SEED = [...]` в `export const METRICS_SEED = [...]`
- Переименовать в `data.js` → `data.js` (можно оставить имя, но это обычный JS-модуль теперь, не в window).
- В `app.jsx` (или где используется) — `import { METRICS_SEED } from './data.js'`

### 4. Импорт CSS

- В entry-point JSX (там, где `createRoot`) — `import './styles.css'`. Vite автоматически подхватит.
- Никаких `?v=25` cache-bust query-strings — это решает Vite на этапе билда через хеши в имени файла.

### 5. Обновление Catalog.html → src/index.html

- Удалить из HTML-файла CDN-теги для React/ReactDOM/Babel — они больше не нужны.
- Удалить `<link rel="stylesheet" href="styles.css">` — теперь CSS приходит через JS-импорт.
- Удалить `<script type="text/babel">` теги — Vite обрабатывает JSX сам.
- Оставить только `<div id="root"></div>` и `<script type="module" src="/path/to/main.jsx"></script>`.

### 6. .gitignore

- Добавить в `.gitignore` корня проекта (если ещё нет): `node_modules/`, `dist/`. **Проверь** существующий `.gitignore` — там уже есть `node_modules/` и `dist/` как исключения. Если есть — не дублируй.

### 7. README в src/ (опционально, короткий)

- Если в `src/` нет README — добавить короткий `src/README.md` с одной фразой: «UI-код прототипа Catalog. Запуск: `npm run dev` из корня проекта.»
- Если уже есть README в корне с инструкцией — не дублировать.

### 8. Точечный фикс читаемости: заголовки карточек метрик

**Это единственное исключение из «не фиксим баги» в этом спринте.** Найден визуальный баг прототипа, мешающий тестированию билда:

- **Симптом:** на странице Catalog в каждой карточке метрики название (например, «GGR Margin / Hold %», «FTD Count», «Uptime», «Total Winnings», «Bonus Cost Ratio») имеет цвет, почти неотличимый от фона карточки. Текст еле виден.
- **Где:** селектор в `styles.css` для заголовка карточки в каталоге (НЕ в drawer-карточке справа — там заголовок «Uptime» виден нормально).
- **Что сделать:** поднять контраст текста заголовка карточки в каталоге до уровня, который читается на тёмном фоне (как заголовок «Uptime» в drawer — там читается). Конкретное цветовое значение — на твоё усмотрение, но не вводи новые CSS-переменные, переиспользуй существующую палитру из `styles.css`.
- **Что НЕ трогать:** drawer-карточка справа — её заголовок виден ок, не менять. Цвета остальных элементов (метрики, графики, кнопки, подзаголовки типа «Revenue · Product») — не менять.
- **Минимум:** одна правка одного CSS-селектора. Если оказывается сложнее — остановись, занеси в `Notes / Tech Debt`, не делай.

**Почему это исключение:** без читаемых заголовков пользователь не сможет полноценно протестировать билд — невозможно отличить карточки, не наводя на каждую. Все остальные косметические/функциональные баги, найденные в процессе работы — НЕ фиксить, заносить в `Notes / Tech Debt` отчёта.

## Testing

После реализации:

1. `npm install` отрабатывает без ошибок
2. `npm run dev` запускает dev-сервер. Браузер по `http://localhost:<port>` показывает каталог метрик в **визуально том же виде**, что был у Catalog.html через `npx serve`.
3. Все ~48 метрик отрендерены
4. Карточка метрики открывается при клике
5. Graph view переключается и работает
6. Tweaks panel (если она была видна) работает
7. Никаких ошибок React в DevTools console (warnings о deprecated key prop / missing keys — допустимы, это унаследовано из прототипа, фиксим в Sprint 2+)
8. `npm run build` собирает `dist/index.html` + JS-бандлы + CSS-бандл
9. `npm run preview` запускает прод-бандл, визуально идентичен dev-сборке

**Скриншот в sprint-report:** один скриншот каталога после `npm run dev` (для подтверждения визуальной идентичности).

## DO NOT

Категорически не делать в этом спринте:

- ❌ **Не править логику компонентов**, не оптимизировать React, не выносить общие куски в хуки. Только синтаксис импортов/экспортов.
- ❌ **Не фиксить баги прототипа** — **кроме задачи 8 (читаемость заголовков карточек)**. Все остальные найденные баги — в `sprint-report-1.md` секцию `## Notes / Tech Debt`, не трогать.
- ❌ **Не подключать новые либы** — никаких lucide-react, recharts, shadcn/ui, Tailwind, classnames, date-fns. Прототип их не использует.
- ❌ **Не мигрировать на TypeScript.** JSX остаётся JSX.
- ❌ **Не трогать CSS** — кроме одного селектора в задаче 8 (заголовки карточек). Всё остальное в `styles.css` копируется/импортируется как есть.
- ❌ **Не трогать `default-vault/_seed/igaming_metrics.json`** — это для будущей миграции (Sprint M2), не для этого спринта. Данные в карточках берутся из `data.js` как было.
- ❌ **Не делать Electron-обёртку.** Только браузерный билд через Vite.
- ❌ **Не настраивать тесты** (Jest, Vitest, Playwright). Тестов сейчас нет — добавим в отдельном спринте.
- ❌ **Не настраивать ESLint/Prettier.** Стиль кода — без изменений.
- ❌ **Не переименовывать компоненты, файлы** (кроме `Catalog.html` → `index.html` если выбран этот путь). Если рефакторинг требует переименования — обоснуй в sprint-report.
- ❌ **Не делать commit** в git до финальной проверки — собирается одним коммитом в конце спринта (или серией атомарных коммитов с понятными сообщениями).
